文本验证操作符说明

{% endif %}

在测试数据中使用#{}#标识文本验证操作符，通常由操作符及操作符参数构成，操作符及参数通过竖线分隔。如不提供参数，则竖线也应去掉，如：#{re}#，#{count|2}#。操作符用于告知程序使用何种方法验证内容，参数用于修饰该方法，具体使用方法见下。
注意：
操作符大小写不敏感，参数为日期格式时大小写敏感，其余大小写不敏感。
验证后将在测试结果返回是否匹配的标识，以及期望值在实际值中匹配的次数等信息。

操作符

  • 未提供
  • ===
  • !==
  • ==
  • !=
  • <=，>=，<，>
  • count
  • encode
  • decode
  • re
  • json
  • xpath
  • replace

如果期望值不含操作符，那么将认为期望值和实际值都是文本（把实际值转换为str类型），然后判断实际值是否包含期望值。

例：
实际值（待验证内容）实际值类型期望值（文本验证表达式）结果
123str123匹配
1234str123匹配
123int123匹配

判断期望值和实际值是否完全相等（类型和值是否都相等）。接受0到2个参数。

无参数

无参数时认为期望值的类型为str，相当于提供了参数str。

例：
实际值（待验证内容）实际值类型期望值（文本验证表达式）结果

1234str#{===}#123不匹配
123int#{===}#123不匹配

有参数

参数代表期望值的类型，比较前会将期望值转换为对应的类型。

参数

  • int

  • time或datetime

验证期望值在实际值中出现的次数是否符合期望，可以和其他操作符嵌套使用。接受0或1参数。

无参数

出现次数大于0次则通过。

例：
实际值（待验证内容）期望值（文本验证表达式）结果
test test1 test2 test3#{count}#test1匹配
test test1 test2 test3#{count}#test4不匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count}##{json}#{"a":1}匹配

有参数

参数为期望的出现次数。参数可以用数字代表固定次数，也可以使用表达式进行高级匹配，在表达式中$[x]$代表实际出现的次数。

例：

test test1 test2 test3#{count|0}#test4匹配
test test1 test2 test3#{count|2}#test1不匹配
test test1 test2 test3#{count|4}#test匹配
test test1 test2 test3#{count|3}##{re}#test\d+匹配
test test1 test2 test3#{count|4}##{re}#test\d+不匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count|2}##{json}#{"a":1}匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count|1}##{json}#{"a":1}不匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count|$[x]$>0 and $[x]$<=2}##{json}#{"a":1}匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count|$[x]$>0 and $[x]$<2}##{json}#{"a":1}不匹配
[{"a":1,"b":"12"},{"a":1,"b":"22"}]#{count|$[x]$>0 or $[x]$<2}##{json}#{"a":1}匹配

对文本进行编码操作并返回操作后的值。接受1或2个参数。

参数

参数1代表编码类型，目前支持的编码方式有："url", "html", "md5", "sha1", "sha224", "sha256"。

参数2用于不同编码类型的特别处理，详见下表。

  • url
  • html
  • （哈希函数）

对文本进行url编码。参数2用于保留不转换的字符。

不提供参数2代表默认情况，即只保留/不转换，其余特殊字符均转换。

none代表全部特殊字符都转换。

save代表保留表达式后的字符，其余特殊字符均转换。

下面的例子的待处理文本为

www.baidu.com/s?ie=UTF-8&wd=abc+d
例：
操作符处理结果
#{encode|url}#www.baidu.com/s%3Fie%3DUTF-8%26wd%3Dabc%2Bd
#{encode|url,none}#www.baidu.com%2Fs%3Fie%3DUTF-8%26wd%3Dabc%2Bd
#{encode|url,save}#/www.baidu.com/s%3Fie%3DUTF-8%26wd%3Dabc%2Bd
#{encode|url,save}#/?=+www.baidu.com/s?ie=UTF-8%26wd=abc+d

对文本进行html编码。参数2用于防止重复编码。

不提供参数2代表自动防止重复编码，即检查到文本已包含html编码就不再进行编码。

force代表关闭重复编码检查，强制进行编码。

例：
待处理文本操作符处理结果
returnUrl=test2?b=2&c=3#{encode|html}#returnUrl=test2?b=2&amp;c=3
returnUrl=test2?b=2&c=3#{encode|html,force}#returnUrl=test2?b=2&amp;c=3
returnUrl=test2?b=2&amp;c=3#{encode|html}#returnUrl=test2?b=2&amp;c=3
returnUrl=test2?b=2&amp;c=3#{encode|html,force}#returnUrl=test2?b=2&amp;amp;c=3

对文本进行解码操作并返回操作后的值。接受1或2个参数。

参数

参数1代表解码类型，目前支持的解码方式有："url", "html"。

参数2用于不同解码类型的特别处理，详见下表。

  • url
  • html
  • jwt

对文本进行url解码。参数2代表是否把+号解码为空格。

不提供参数2代表默认情况，即保留+号。

plus代表把+号解码为空格。

www.baidu.com/s?ie=UTF-8%26wd=abc+d#{encode|url,plus}#www.baidu.com/s?ie=UTF-8&wd=abc d

有参数

参数1

参数1代表正则匹配规则。见下表。

匹配规则表（参数对大小写不敏感）：
参数匹配规则
I使匹配对大小写不敏感
L做本地化识别（locale-aware）匹配
M多行匹配，影响^和$
S使.匹配包括换行在内的所有字符
U根据Unicode字符集解析字符。这个标志影响\w，\W，\b，\B
X给予更灵活的格式以便将正则表达式写得更易于理解
例：
实际值（待验证内容）实际值类型期望值（文本验证表达式）结果

ab\ncstr#{re|S}#AB.C不匹配
ab\ncstr#{re|SI}#AB.C匹配
ab\ncstr#{re|IS}#AB.C匹配
ab\ncstr#{re|is}#AB.C匹配

参数2

参数2代表是否返回正则表达式匹配到的值，0代表返回第一个匹配的值，1代表第二个，以此类推。

例：
实际值（待验证内容）实际值类型期望值（文本验证表达式）结果返回值
vic.wang@test.comstr#{re|,0}#vic\.wang@匹配vic.wang@
vic.wang@test.comstr#{re|,0}#vic\.(.*)@匹配wang
Vic.wang@test.comstr#{re|,0}#vic\.(.*)@不匹配
Vic.wang@test.comstr#{re|I,0}#vic\.(.*)@匹配wang
vic.wang@123.com;abc@456.comstr#{re|,0}#([\w.-]+)(?=@)匹配vic.wang

把期望值和实际值视为json字符串，双方都转换为对象后进行匹配。接受0或多个参数。

json字符串必须是符合规定格式的字符串，如：key必须为文本且使用双引号标识，value可以为文本或数字，为文本时需使用双引号标识。

期望值转换得出的对象，下称期望对象；实际值转换得出的对象，下称实际对象。

对象中的值如果是str类型，则会被用于新一轮文本验证，可以嵌套使用文本验证操作符。

无参数

期望对象存在于实际对象中，且相对的层级结构一致，则匹配成功。

对文本进行替换操作并返回操作后的值。接受1到4个参数。

参数1

一个字符串，代表需要替换的字符。

参数2

一个字符串，代表替换后的字符，不提供或留空则删除需要替换的字符。

参数3

一个正整数，代表替换几次，不提供则全部替换；默认为从左向右替换，如需从右向左，请使用参数4。

参数4

可以是任意值，例如数字1，只要有值就代表从右向左替换。

注意

参数1及参数2的字符串如果需要使用逗号，需写作反斜杠逗号\,，如需使用反斜杠，则应写作2个反斜杠\\。
