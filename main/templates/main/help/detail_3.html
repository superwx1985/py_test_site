文本验证连接符说明

{% endif %}

在测试数据中使用#&&#表示两侧文本验证条件是且的关系，当两侧条件都满足时，验证通过。

在测试数据中使用#||#表示两侧文本验证条件是或的关系，当两侧条件任一满足时，验证通过。

当两种连接符同时使用时，且的优先级大于或。

关系判断不适用短路规则，所有条件都会被判断，然后才返回结果。

例：
实际值期望值结果
test test1 test2 test3test1#&&#test2匹配
test test1 test2 test3test1#&&#test4不匹配
test test1 test2 test3test1#||#test4匹配
test test1 test2 test3test1#||#test4#&&#test5匹配
test test1 test2 test3test1#&&#test4#||#test5不匹配
test test1 test2 test3test1#&&#test3#||#test1#&&#test4匹配
[{"a":1,"b":"12"},{"a":2,"b":"22"}]#{json|l}#[{"a":1},{"a":2}]#||##{json|l}#[]匹配
[{"a":1,"b":"12"},{"a":2,"b":"22"}]#{json|l}#[{"a":1},{"a":2}]#&&##{json|l}#[]不匹配
[{"a":1,"b":"12"},{"a":2,"b":"22"}]#{json|l}#[{"a":1},{"a":2}]#&&##{json|l}#[]#||##{json}#[]匹配
[{"a":1,"b":"12"},{"a":2,"b":"22"}]#{json|l}#[{"a":1},{"a":2}]#&&##{json|l}#[]#||##{json|l}#[]不匹配
